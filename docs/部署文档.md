### 6.6 spark-submit 参数说明

consumer-start.sh 内部执行的 spark-submit 命令:

```bash
spark-submit \
  --class com.stock.streaming.QuoteStreamingJob \
  --master spark://master0:7077 \
  --deploy-mode client \
  --executor-memory 2G \
  --total-executor-cores 6 \
  --driver-class-path "/root/stock-app:/root/stock-app/mysql-connector-java-5.1.47.jar" \
  --jars /root/stock-app/mysql-connector-java-5.1.47.jar \
  --conf spark.streaming.kafka.maxRatePerPartition=2000 \
  /root/stock-app/stock-bigdata-core-1.0-SNAPSHOT.jar \
  [--replay] [--date YYYY-MM-DD]
```

| 参数 | 说明 | 默认 |
|------|------|------|
| `--executor-memory` | 每个 Executor 内存 | 2G |
| `--total-executor-cores` | 所有 Executor 总核数 | 6 |
| `--replay` | 从头消费 (Kafka offset 归零) | 关闭 |
| `--date YYYY-MM-DD` | 只处理指定日期 | 全部 |
| `maxRatePerPartition` | 每分区每批最大记录数 | 2000 |
| `--driver-class-path` | Driver 额外 classpath (必须含 `/root/stock-app/`) | — |

### 6.7 停止消费端

```bash
bash /root/stock-app/scripts/consumer-stop.sh
```

停止逻辑: 创建 `/tmp/stock-consumer-stop` 标记文件 → Consumer 检测到 → flush Redis → MySQL → 优雅退出 (60s 超时后强制 kill)。

### 6.8 停止集群

```bash
bash /root/stock-app/scripts/cluster-stop.sh
```

**停止顺序**: Spark → YARN → HDFS → MySQL → Redis → Kafka → ZK (与服务启动顺序相反)

### 6.9 断点续跑

若需中途停止后继续消费:

```bash
# 1. 停止前确保 feature.redis.flushdb.onstart=false
sed -i 's/feature.redis.flushdb.onstart=true/feature.redis.flushdb.onstart=false/' \
  /root/stock-app/application.properties

# 2. 正常停止
bash /root/stock-app/scripts/consumer-stop.sh

# 3. 下次启动时使用正常模式 (不加 --replay)
bash /root/stock-app/scripts/consumer-start.sh
```

Kafka offset 自动从上次提交位恢复。

## 7. 第 5 步：启动全链路回放

> **操作节点**: master0 (192.168.137.201)
>
> **说明**: `full-replay.sh` 一次消费全部 Kafka 数据，日清自动分天 flush MySQL。适合批量回放历史数据。

### 7.1 全量回放

```bash
ssh root@192.168.137.201
cd /root/stock-app/scripts

# 一键全量回放
nohup bash full-replay.sh > /tmp/replay.log 2>&1 &
```

**工作流程**:

```
1. cluster-clean.sh --kafka --no-mysql  → 清 Redis + HDFS，重置 offset
2. consumer-start.sh --replay           → 从 0 开始消费全部
3. 轮询 Kafka lag → 等待消费完毕        → 无超时持续等待
4. consumer-stop.sh                     → 优雅停止 (shutdown flush)
5. FlushJob                             → 兜底 flush 最后一天
```

### 7.2 逐日回放 (推荐)

如果 Kafka 数据跨天交错，推荐逐日处理:

```bash
bash daily-replay.sh 2026-07-02 2026-07-03 2026-07-06
```

### 7.3 监控回放进度

```bash
# 查看系统状态 (Redis 实时写入)
redis-cli -a <REDIS_PASSWORD> -h 192.168.137.210 HGETALL stock:system:status

# 关键字段
# status: running / flushing / error
# batch_count: 累计 batch 数
# current_date: 当前处理到哪个交易日
# consumer_lag / consumer_pct: Kafka 消费进度
# flush_windows_done / flush_windows_total: 归档进度 (状态为 flushing 时有效)
```

### 7.4 手动 Flush (Redis → MySQL)

```bash
source /etc/profile

java -cp "/root/stock-app/mysql-connector-java-5.1.47.jar:/root/stock-app/stock-bigdata-core-1.0-SNAPSHOT.jar" \
  com.stock.batch.FlushJob [--date 2026-07-02]
```
